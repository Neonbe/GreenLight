## 一、重构动机

### 1.1 现状问题

当前 `PopoverView.swift` 是为 **Menu Bar Popover (420×380)** 设计的紧凑面板。在转为 **Dock App 主窗口** 后，同一套布局直接套用在大窗口中，产生以下问题：

| # | 问题 | 严重度 |
|---|------|--------|
| P1 | **窗口与 Onboarding 尺寸不一致** — Onboarding 900×620，主界面 420×380 | 高 |
| P2 | **空间浪费** — 黄灯区仅扫描时出现，大部分时间只有红+绿两个 Lane | 高 |
| P3 | **信息密度与窗口比例失衡** — 少量内容挤在小区域，其余全是空白 | 高 |
| P4 | **字体风格不统一** — 主界面仍用 `.bold/.heavy`，与 Onboarding `.light` 风格断裂 | 中 |
| P5 | **标题栏风格不统一** — 主界面仍显示系统默认标题栏 | 中 |

### 1.2 设计目标

| 目标 | 验收标准 |
|------|----------|
| **空间利用** | 三区（红/黄/绿）始终可见，无论数据多少都不浪费空间 |
| **视觉一致** | 主界面与 Onboarding 风格统一：900×620、hiddenTitleBar、`.light` 字体 |
| **信息层次** | 一眼看到全局状态，按需下钻到具体 App |
| **品牌强化** | 红绿灯作为视觉锚点始终在位 |

## 二、布局方案

### 2.1 整体架构

```
┌──────────────────────────────────────────────────────────────────┐
│ ●●●                                                              │
│                                                                  │
│    G R E E N L I G H T                              ↻    ⚙     │
│                                                                  │
│    ┌────────────────────────────────────────────────────────┐    │
│    │                                                        │    │
│    │              ┌────────┐                                │    │
│    │              │        │                                │    │
│    │              │  🔴 ─────── Red Lane ────────────────  │    │
│    │              │        │                                │    │
│    │              │  🟡 ─────── Amber Lane ──────────────  │    │
│    │              │        │                                │    │
│    │              │  🟢 ─────── Green Lane ──────────────  │    │
│    │              │        │                                │    │
│    │              └──┬─────┘                                │    │
│    │                 │                                      │    │
│    └────────────────────────────────────────────────────────┘    │
│                                                                  │
│                         ● System Protected                       │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 核心设计决策

| 决策 | 说明 |
|------|------|
| **三区始终可见** | 红、黄、绿 Lane 永远显示，无数据时展示空状态文案，避免布局跳动 |
| **红绿灯居左，Lane 水平延伸** | 沿用 `ui_design_spec.md §3.1` 的锚点布局，放大至主窗口比例 |
| **Lane 等高分布** | 三个 Lane 等分内容区高度，保证视觉均衡 |
| **统一字体风格** | 品牌名、标签、数字全部采用 `.light/.regular` 字重，与 Onboarding 一致 |
| **居中布局** | 内容块整体居中，左右留白各 ~80px，上下留白充足 |

### 2.3 Lane 空状态文案

| Lane | 有数据 | 空状态 |
|------|--------|--------|
| 🔴 Red | `BLOCKED  {N}` + App 图标 | `No blocked apps` |
| 🟡 Amber | `SCANNING  {N}` + App 图标 | `Monitoring…` |
| 🟢 Green | `CLEARED  {N}` + App 图标 | `All clear` |

## 三、视觉规格

### 3.1 窗口

| 属性 | 值 | 说明 |
|------|-----|------|
| 尺寸 | 900 × 620 | 与 Onboarding 一致 |
| 标题栏 | `.windowStyle(.hiddenTitleBar)` | 已在 GreenLightApp.swift 中生效 |
| 背景 | `#0F172A` + 微妙径向渐变 | 复用 Onboarding 背景 |

### 3.2 字体

| 元素 | 字重 | 字号 | 颜色 |
|------|------|------|------|
| 品牌名 `G R E E N L I G H T` | `.light` | 14px | `rgba(248,250,252,0.25)` |
| Lane 标签 `BLOCKED` | `.light` | 10px, tracking 2 | `rgba(248,250,252,0.3)` |
| Lane 计数 | `.light` | 28px | 对应灯色 |
| 空状态文案 | `.light` | 12px | `rgba(248,250,252,0.2)` |
| 状态胶囊 | `.light` | 11px | `#22C55E` 40% |

### 3.3 红绿灯

复用 Onboarding 的 `OBTrafficLight` 组件，放大至主界面尺寸：

| 属性 | Onboarding | 主界面 |
|------|-----------|--------|
| 灯泡直径 | 30px | 40px |
| 外壳宽度 | 52px | 68px |
| 外壳高度 | 120px | 160px |

### 3.4 Lane

| 属性 | 值 |
|------|-----|
| 高度 | 等分内容区，每个约 120px |
| 左侧霓虹线 | 3px, 灯色 + `shadow blur 6` |
| 背景 | `rgba(255,255,255,0.03)` |
| 边框 | `1px solid rgba(255,255,255,0.04)` |
| 圆角 | 14px（右侧）/ 0（左侧接灯体） |
| 内边距 | 20px |

### 3.5 状态胶囊

| 属性 | 值 |
|------|-----|
| 位置 | 底部居中 |
| 绿点 | 5px, `#22C55E`, 呼吸闪烁 3s |
| 文字 | "System Protected" / "累计亮绿灯 N 次" |
| 风格 | `.light` 11px, green 40% |

## 四、交互细节

### 4.1 App 图标

沿用现有 `ui_design_spec.md §3.3` 的交互规范：
- Hover: `scale(1.15) translateY(-2px)` + 阴影加深
- Click: 弹出 ActionBubble (popover)
- 交错入场: 每个图标 30ms 延迟

### 4.2 扫描按钮

- 位置：Header 右侧 `↻` 图标
- 点击时：黄灯区显示 scanning 状态
- 扫描完成：黄灯区如无等待项则显示 "Monitoring…"

### 4.3 Expand Coverage 提示

- 位置：底部状态栏上方
- 样式：蓝色底卡片，与当前一致
- 触发条件：扫描后发现未授权目录

## 五、双入口复用

| 入口 | 视图 | 说明 |
|------|------|------|
| 主窗口 (WindowGroup) | 完整 900×620 主界面 | Dock App 的核心入口 |
| Menu Bar (MenuBarExtra) | 紧凑版 PopoverView | 保持现有 420×380 紧凑布局 |

> [!IMPORTANT]
> 主窗口和 Menu Bar Popover 使用**同一份数据 (AppState)**，但**布局独立**。Menu Bar 保持紧凑，主窗口充分利用空间。
