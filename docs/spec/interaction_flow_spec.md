# GreenLight：交互流程定稿

> **产品名称**：GreenLight（原 Gatekeeper Companion）  
> **核心隐喻**：🚦 红绿灯——为你的应用"放行"

---

## 一、红绿灯设计语言

GreenLight 的整体设计围绕**交通信号灯**的隐喻展开：

| 信号 | 含义 | 状态说明 |
|------|------|---------|
| 🟢 **绿灯** | 已放行 | 被 GreenLight 修复过的应用。记录"绿灯"放行次数（含同一 app 更新后的重复放行） |
| 🔴 **红灯** | 被拦截 | 当前被 Gatekeeper 拦截、等待处理的应用。提示风险 |
| 🟡 **黄灯** | 等待中 | 刚检测到、正在扫描/等待用户决策的应用 |

**品牌延伸**：
- Menu Bar 图标：常态为 🟢（一切正常）；检测到拦截时变 🔴 + 角标
- 通知文案："GreenLight 检测到一个被拦截的应用" → "已为 XXX 亮绿灯 ✅"
- 仪表盘统计："已放行 42 个应用，累计亮绿灯 67 次"（含更新后重复放行）
- app 更新后重新触发隔离 → "XXX 更新后需要重新放行" → 体现**持续保护价值**

## 二、确认的交互模型：A + C 融合

### 主路径（模型 A：反应式守护）

被拦截时**自动推送通知**，三个操作按钮：

```
┌────────────────────────────────────────┐
│  🚦 GreenLight                         │
│                                        │
│  macOS 拦截了 Antigravity Tools        │
│                                        │
│  [❌ 忽略]  [🔓 修复]  [🛠 修复并打开]  │
└────────────────────────────────────────┘
```

| 按钮 | 行为 |
|------|------|
| **忽略** | 关闭通知，不做任何操作。记录但不处理 |
| **修复** | 仅执行 `xattr -rd`，解除隔离。不打开 app |
| **修复并打开** | 执行 `xattr -rd` + 自动启动 app |

> 通知之外，也可以在 **Menu Bar 弹出面板 (Popover)** 中展示同样的信息，解决"通知横幅消失太快"的问题。

### 兜底路径（模型 C：Menu Bar 仪表盘）

用户错过通知时，通过 Menu Bar 角标 + 弹出面板查看和处理：

```
┌──────────────────────────────────────┐
│  🚦 GreenLight                       │
│                                      │
│  🔴 被拦截 (1)                       │
│  ├─ Antigravity Tools                │
│  │  /Applications/...                │
│  │  [🔓 修复]  [🛠 修复并打开]        │
│  │                                   │
│  🟢 已放行 (12)                      │
│  ├─ Some App  · 绿灯 ×3 · 2小时前    │
│  ├─ Another   · 绿灯 ×1 · 昨天       │
│  └─ ...                              │
│                                      │
│  ─────────────────                   │
│  累计亮绿灯 42 次                     │
│  ⚙️ 设置  |  📊 全盘扫描             │
└──────────────────────────────────────┘
```

### 首次安装路径

解决"鸡生蛋"悖论——全盘扫描即时展现价值：

```
┌──────────────────────────────────────┐
│  🚦 欢迎使用 GreenLight              │
│                                      │
│  正在扫描已安装应用...               │
│                                      │
│  发现 2 个被隔离的应用：              │
│  🔴 Antigravity Tools               │
│  🔴 Some Other App                  │
│                                      │
│  [🟢 全部放行]  [逐个查看]            │
└──────────────────────────────────────┘
```

## 三、持续保护价值

> [!IMPORTANT]
> **已修复的应用在版本更新后会重新被隔离。** 这意味着 GreenLight 不是一次性工具——每次 app 更新，用户都需要重新"放行"。这构成了**持续的保护价值**和用户粘性。

```
Antigravity Tools v1.0 → 🟢 已放行
         ↓ (用户更新到 v1.1)
Antigravity Tools v1.1 → 🔴 被重新隔离
         ↓ (GreenLight 自动检测)
通知："Antigravity Tools 更新后需要重新放行"
         ↓ (用户一键修复)
Antigravity Tools v1.1 → 🟢 已放行（绿灯 ×2）
```

## 四、设计难题确认结论

| # | 难题 | 结论 |
|---|------|------|
| 1 | 通知时机 | ✅ 通知在弹窗消失后仍可通过 **Menu Bar 面板**查看。双通道：系统通知 + Menu Bar Popover |
| 2 | 用户点了"移到废纸篓" | ✅ 简单提示"应用已被移走，请先恢复"。MVP 不处理自动恢复 |
| 3 | 操作按钮设计 | ✅ 三按钮：**忽略 / 修复 / 修复并打开** |
| 4 | 信任悖论 | ✅ 不是问题，通过 Onboarding 和清晰的通知文案自然解决 |
| 5 | 通知疲劳 | ✅ 不是问题，反应式模型只在真正被拦截时触发 |
| 6 | 首次安装鸡生蛋 | ✅ 全盘扫描解决 |

## 五、完整用户旅程

```
┌─ 首次安装 ─────────────────────────────────────────────────┐
│  安装 GreenLight → 首启 Onboarding → 授权通知             │
│  → 全盘扫描 → 展示已有隔离 app → [全部放行] / [逐个查看]  │
│  → 设置开机自启 → Menu Bar 🟢 常驻                        │
└────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─ 日常守护 ────────────────────────────────────────────────┐
│                                                           │
│  Menu Bar 🟢 安静待命                                     │
│                                                           │
│  用户双击未签名 app → Gatekeeper 弹窗                     │
│     │                                                     │
│     ├── Menu Bar 🟢→🔴 变色 + 角标                        │
│     ├── 系统通知 / Menu Bar Popover                       │
│     │   [❌ 忽略] [🔓 修复] [🛠 修复并打开]                │
│     │                                                     │
│     ├─ 路径 1: 点通知 [修复并打开] → 🟢 搞定              │
│     ├─ 路径 2: 错过通知 → 点 Menu Bar → 手动修复          │
│     └─ 路径 3: 点了"移到废纸篓" → 提示先恢复              │
│                                                           │
│  App 更新后重新隔离 → 自动检测 → 再次通知 → 再次放行      │
│  "绿灯 ×N" 累计                                          │
│                                                           │
└───────────────────────────────────────────────────────────┘
```
