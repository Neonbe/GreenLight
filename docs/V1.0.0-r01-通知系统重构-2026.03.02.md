## 一、背景与动因

### 问题现状

V1.0.0 MVP 实现中，通知系统依赖 macOS 原生 `UserNotifications` 框架。经实测发现**通知完全不工作**，系统通知设置中找不到 GreenLight。排查发现多个 bug 叠加：

| # | Bug | 影响 |
|---|-----|------|
| 1 | Onboarding 创建临时 `NotificationManager()` 实例，覆盖 App 级实例的 `delegate` | 通知回调失效 |
| 2 | `requestAuthorization()` 结果被 `_ =` 丢弃 | 权限拒绝无降级 |
| 3 | `setupPipeline()` 在 `init()` 中执行，先于 Onboarding 权限请求 | 事件触发时可能无权限 |
| 4 | 无 App 内通知机制 | 无权限时完全静默 |

### 决策

**不修补系统通知的 bug，而是换一个更优的架构**：用自定义浮动面板（`NSPanel`）替代系统通知作为主通知通道。

**核心理由**：
1. **零权限依赖** — 自定义面板不需要任何系统通知权限，即装即用
2. **体验更好** — 可直接嵌入 App 图标、操作按钮、自定义动画，比系统通知更丰富
3. **场景契合** — Gatekeeper 弹窗出现时用户必然在电脑前，不需要系统通知的"离线推送"能力
4. **Onboarding 简化** — 不再需要通知权限授权步骤，降低用户摩擦

## 三、功能规格

### 3.1 检测浮动面板（替代 F5）

**触发条件**：EventDeduplicator 输出 GreenLightEvent 时自动弹出。

#### 3.1.1 视觉规格

面板复用项目 `ui_design_spec.md` 的设计系统（色彩 Token、字体、质感层次），保持视觉统一。

**面板结构**：

```
┌──────────────────────────────────────────────┐
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │  [ StatusIndicator ]                   │  │
│  │    ● 检测到拦截                         │  │
│  │                                        │  │
│  │  ┌──────┐                              │  │
│  │  │ Icon │  AppName                     │  │
│  │  │ 40px │  /Applications/...           │  │
│  │  └──────┘                              │  │
│  │                                        │  │
│  │  ┌──────────────────────────────────┐  │  │
│  │  │ [忽略]   [修复]   [修复并打开]    │  │  │
│  │  └──────────────────────────────────┘  │  │
│  │                                        │  │
│  │  ─── 10s ━━━━━━━━━━━━━━━━━━━ ───     │  │  ← 倒计时进度条
│  └────────────────────────────────────────┘  │
│                                              │
└──────────────────────────────────────────────┘
```

**视觉属性**：

| 属性 | 值 | 来源 |
|------|-----|------|
| 面板宽度 | 320px | — |
| 面板圆角 | 16px | 介于 Popover 28px 和 Lane 12px 之间 |
| 背景 | `rgba(22,28,45,0.95)` + `backdrop-filter: blur(80px) saturate(1.6)` | 复用 `ui_design_spec` 质感层次 |
| 外阴影 | `0 20px 60px rgba(0,0,0,0.4)` | 略轻于 Popover（面板更小更轻） |
| 内高光 | `inset 0 1px 0 rgba(255,255,255,0.06)` | 统一毛玻璃高光 |
| 底暗边 | `inset 0 -1px 0 rgba(0,0,0,0.1)` | 统一 |
| 边框 | `1px solid rgba(255,255,255,0.08)` | `--glass-border` |
| 内边距 | 20px | — |

**状态指示灯**：

| 属性 | 值 |
|------|-----|
| 圆点尺寸 | 8px |
| 颜色 | `--red` (#EF4444)，叠加 `box-shadow: 0 0 8px rgba(239,68,68,0.6)` 霓虹辉光 |
| 脉冲动画 | `redPulse` 2.5s ease-in-out infinite（复用现有红灯动画） |

**App 信息区**：

| 元素 | 字体 | 颜色 |
|------|------|------|
| App 图标 | 40×40px, 圆角 10px | — |
| App 名称 | Inter 600, 14px | `--text-primary` (#F8FAFC) |
| App 路径 | Inter 400, 11px | `--text-muted` (rgba(248,250,252,0.25)) |

**按钮区**：

| 按钮 | 背景 | 文字 | 圆角 |
|------|------|------|------|
| 忽略 | `rgba(255,255,255,0.06)` | `--text-secondary` | 10px |
| 修复 | `rgba(255,255,255,0.08)` | `--text-primary` | 10px |
| 修复并打开 | `--green` (#22C55E) | White | 10px |
| 通用 Hover | `brightness(1.15)` | — | 250ms transition |
| 通用 Active | `scale(0.96)` | — | 150ms |
| Focus-visible | 2px green outline, 3px offset | — | — |

**倒计时进度条**：

| 属性 | 值 |
|------|-----|
| 高度 | 2px |
| 颜色 | `--text-muted` → `--red`（最后 3 秒变红提醒即将消失） |
| 动画 | 宽度从 100% 线性缩减到 0%，总时长 10s |

#### 3.1.2 动画规格

复用 KI 中 `NDAnimation` spring 物理预设，保持全 App 动画一致性。

**入场动画**（面板弹出）：

```swift
// Phase 1: 面板容器
opacity:    0 → 1
translateY: -8px → 0px     // 从上方微滑入（Menu Bar 方向）
scale:      0.96 → 1.0
timing:     NDAnimation.panelTransition  // spring(response: 0.35, dampingFraction: 0.85)

// Phase 2: 内容交错入场（延迟 100ms）
// 状态指示灯
opacity:    0 → 1
delay:      0.1s

// App 图标 + 信息
opacity:    0 → 1
translateX: -6px → 0px
delay:      0.15s
timing:     NDAnimation.cardEnter  // spring(response: 0.25, dampingFraction: 0.6)

// 按钮区
opacity:    0 → 1
translateY: 4px → 0px
delay:      0.2s
timing:     NDAnimation.cardEnter
```

**退场动画**（面板消失）：

```swift
opacity:    1 → 0
translateY: 0 → -4px      // 往上收回
scale:      1.0 → 0.98
timing:     NDAnimation.cardExit  // spring(response: 0.25, dampingFraction: 1.0)
// 无弹跳，干净利落
```

**修复成功动画**：

```swift
// Step 1: 按钮 → 成功状态（200ms）
button.background:  --green → --green (brightness +10%)
button.label:       "修复并打开" → "✓"

// Step 2: 面板整体 SuccessPulse（300ms）
scale:    1.0 → 1.02 → 1.0
brightness: 0 → 0.05 → 0
timing:     NDAnimation.successPulse  // spring(response: 0.5, dampingFraction: 0.6)

// Step 3: 面板优雅退场（延迟 600ms）
// 使用标准退场动画
```

**修复失败动画**：

```swift
// ErrorShake — 水平振动
offset.x:  0 → 10 → -8 → 6 → -4 → 2 → 0
interval:  50ms per step
timing:     NDAnimation.errorShake  // spring(response: 0.1, dampingFraction: 0.2)
// 面板不消失，显示错误提示
```

**自动消失动画**（10 秒超时）：

```swift
// 最后 3 秒进度条变红（warning）
progressBar.color: --text-muted → --red
timing: 250ms ease

// 超时: 使用标准退场动画
```

#### 3.1.3 交互规格

**窗口行为**：

| 属性 | 规格 | 理由 |
|------|------|------|
| 窗口类型 | `NSPanel`，`.nonactivatingPanel` | 不抢焦点 |
| 窗口层级 | `.floating` | 浮于其他窗口之上 |
| 定位锚点 | Menu Bar `NSStatusItem.button?.window?.frame` 正下方居中 | 与 Menu Bar 图标视觉关联 |
| 点击面板外 | 面板不消失（与通知行为一致，需主动操作或等待超时） | 避免意外关闭 |
| 自动消失 | 10 秒无交互后退场 | UX 建议: Toast 3-5s，此处因含操作按钮延长到 10s |
| 用户交互后 | 取消自动消失倒计时（用户在操作中不应被打断） | — |
| ESC 键 | 相当于点击"忽略" | 键盘可达性 |
| 多事件队列 | 新事件到达时：如面板已显示，更新内容（含切换动画）；不叠加多个面板 | 避免屏幕混乱 |

**多事件切换动画**：

```swift
// 当新事件到达且面板已显示时
// Phase 1: 当前内容退出
content.opacity:    1 → 0
content.translateX: 0 → -20px
timing: 200ms ease-out

// Phase 2: 新内容进入
content.opacity:    0 → 1
content.translateX: 20px → 0
timing: 250ms NDAnimation.cardEnter
// 面板容器本身不动，只切换内容
```

**无障碍**：

| 规则 | 实现 |
|------|------|
| `prefers-reduced-motion` | `@Environment(\.accessibilityReduceMotion)` — 禁用所有弹跳和交错，改为瞬时 opacity 渐变 |
| 键盘导航 | Tab 可聚焦三个按钮，Enter 执行 |
| VoiceOver | 面板弹出时读出："GreenLight 检测到 [App名] 被拦截" |
| 对比度 | 所有文字 ≥ 4.5:1（已由 `ui_design_spec` 色彩系统保证） |

### 3.2 Onboarding 简化（修改 F10）

从 3 步流程简化为 2 步：

```
变更前: 欢迎页 → 通知权限页 → 扫描页
变更后: 欢迎页 → 扫描页
```

- 欢迎页的"开始设置"按钮直接跳到扫描页并启动扫描
- 删除通知权限请求页面及相关代码

### 3.3 系统通知降级为可选

- `NotificationManager` 改为单例，保留代码但不作为主通道
- 发送前检查权限：有权限才发，无权限静默跳过
- 未来可在设置页提供"同时发送系统通知"开关（本次不实现）

## 四、技术方案

### 4.1 架构变更

```
变更前:
  EventDeduplicator → NotificationManager (UserNotifications) → 系统通知
                   → AppState → Menu Bar Popover

变更后:
  EventDeduplicator → DetectionPanelController (NSPanel) → 自定义浮动面板
                   → AppState → Menu Bar Popover
                   → NotificationManager (可选，有权限才发)
```

### 4.2 新增模块

| 模块 | 文件 | 职责 |
|------|------|------|
| `DetectionPanelController` | `Notification/DetectionPanelController.swift` | NSPanel 生命周期管理（创建、定位、显示、隐藏、自动消失） |
| `DetectionPanelView` | `Views/DetectionPanelView.swift` | 面板 SwiftUI 视图（App 信息 + 三按钮） |

### 4.3 修改模块

| 模块 | 文件 | 变更 |
|------|------|------|
| `GreenLightApp` | `GreenLightApp.swift` | 管线输出从 NotificationManager 切换到 DetectionPanelController |
| `OnboardingView` | `Views/OnboardingView.swift` | 删除权限页，3 步→2 步 |
| `NotificationManager` | `Notification/NotificationManager.swift` | 单例化，降级为可选增强 |
| `AppState` | `AppState.swift` | 增加 `pendingPanelEvent` 状态 |

## 五、用户旅程变更

### 5.1 日常守护（主路径变更）

```
变更前:
  检测到拦截 → 系统通知推送 → 用户点通知按钮 → 修复

变更后:
  检测到拦截 → Menu Bar 图标下方弹出浮动面板 → 用户点面板按钮 → 修复
```

### 5.2 错过通知（兜底路径不变）

```
面板消失 → 用户点击 Menu Bar 🔴 图标 → Popover 中查看和修复
```
