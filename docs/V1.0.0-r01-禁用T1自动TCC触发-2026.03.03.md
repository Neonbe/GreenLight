# 修复：禁用 T1 自动 TCC 触发

## 问题

App 启动后，用户没有做任何操作，macOS 自动弹出 "GreenLight would like to access files in your Downloads folder" 权限弹窗。

### 根因

`EnhancePromptManager` 的 T1（被动触发）路径在 GK 窗口到期时调用 `canShowPrompt(.t1)` → `checkDirectoryAccess(~/Downloads)` → `FileManager.contentsOfDirectory(at: ~/Downloads)` → **触发 TCC 弹窗**。

```
onGKActivity              ← setupPipeline 注入回调
  → recordGKActivity()    ← 记录 GK 活动到窗口
  → (10s 窗口到期)
  → evaluateGKWindow()
  → count≥2 && !hasDetection → canShowPrompt(.t1)
  → L130: level1Directories.filter { !checkDirectoryAccess($0) }
  → canAccessDirectory(~/Downloads)
  → fm.contentsOfDirectory(at: ~/Downloads)  ← 🔴 TCC 弹窗
```

### 触发条件

只要 LogStream 在 10s 窗口内检测到 ≥2 次 GK 日志（`evaluateScanResult`），且 Channel A 未成功提取路径（`hasDetection = false`），就会自动走到 `canShowPrompt` → `checkDirectoryAccess`。这个条件非常容易满足——用户双击任何 app 都可能触发。

## 修复方案

**最小改动：断开 T1 数据源，保留 T2 手动路径。**

## T2 手动路径验证

确认断开 T1 后，用户仍可通过以下路径手动扩展扫描范围：

```
用户点击 Dashboard "Grant Access" 按钮
  → enhanceManager.requestEnhance()
  → canShowPrompt(forTrigger: .t2)      ← T2 不受冷却限制，始终返回 true
  → attemptUpgradeToLevel1()
  → checkDirectoryAccess(~/Downloads)   ← TCC 弹窗（用户预期内）
  → 用户点 Allow
  → Persistence.level1Granted = true    ← 持久化
  → onUpgradeToLevel1([~/Downloads])    ← 追加 FSEvents 监控
```

重启后恢复：

```
setupPipeline()
  → Persistence.level1Granted == true
  → fsWatcher.addDirectories(level1Directories)   ← 自动恢复
  → warmupScan() 范围包含 Downloads                ← 已有 TCC 授权，不弹窗
```
