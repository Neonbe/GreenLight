## 一、r04 复盘：为什么失败了

### 1.1 致命问题：日志全部被脱敏

macOS `os_log` 框架的隐私策略：**Swift `Logger` 中 `String` 类型的字符串插值默认标记为 `<private>`**。通过 `log stream` 命令收集日志时，所有非标量参数都显示为 `<private>`。

**直接后果**：整个 r04 实验中，所有 APP 身份信息（名称、路径、bundleId、syspolicyd 原始日志内容）都被遮蔽，无法回答核心问题。

### 1.2 其他缺陷

| 缺陷 | 描述 |
|------|------|
| 步骤间隔不足 | 方案要求 30s 间隔，实际多处不足，导致事件重叠 |
| USER_MARK 无编号 | 所有 USER_MARK 格式一致，只能靠时间推断对应哪个步骤 |
| 对照组不合格 | SubLime 实际被 rejected，并非真正的 normal app 对照 |
| 信息维度不足 | 缺少签名详情、Quarantine 来源等关键信息 |

## 二、实验目标（r07 升级版）

### 核心问题

> **不同类型的 APP 在管线各阶段分别报出哪些信息？这些信息是否足以确认「当前被 GK 拦截的是谁」？**

### 具体子问题

1. **身份确认**：每种 APP 类型，我们能获取到哪些身份标识（名称、路径、bundleId、签名 identifier、teamId）？
2. **GK 事件模式**：双击 blocked app 产生的 syspolicyd 事件序列（明文），与打开 normal app 有何差异？
3. **早期信号**：GK 事件中是否包含被检查 APP 的路径/bundleId，使我们能在 FSEvents 到达前就知道「是谁」？
4. **签名分类**：不同签名状态（未签名 / ad-hoc / 篡改 / 已公证）的 SecStaticCode 返回什么信息？
5. **Quarantine 溯源**：quarantine xattr 中包含的来源信息（下载者、URL）对用户诊断有多大帮助？

## 三、代码改动清单

> ⚠️ **所有改动仅用于 DEBUG 实验，Release 构建不受影响。**

### 3.1 消除 `<private>` 遮蔽

**原理**：Swift `Logger` API 中，对字符串插值添加 `privacy: .public` 标记：

```swift
// 修改前（被脱敏）：
GLLog.logStream.debug("GK line: \(line)")

// 修改后（明文显示）：
GLLog.logStream.debug("GK line: \(line, privacy: .public)")
```

**需修改的文件**：

| 文件 | 需修改行数 | 内容 |
|------|----------|------|
| `LogStreamMonitor.swift` | ~12 行 | GK line、GK scan、GK evaluate、GK event、GK timeline、Detected |
| `FSEventsWatcher.swift` | ~8 行 | FS event、FSEvents detected、Quarantine check、Fallback scan |
| `GatekeeperAssessor.swift` | ~7 行 | SecStaticCode CREATE/VALIDITY、GK assess、spctl |
| `EventDeduplicator.swift` | ~3 行 | Dedup new、Dedup merge、Dedup flush |
| `GreenLightApp.swift` | ~6 行 | FS↔GK correlation、Pipeline received、Showing panel、USER_MARK |

### 3.2 新增：签名详情提取

在 `GatekeeperAssessor.swift` 的 `quickReject()` 中，当 `SecStaticCodeCreateWithPath` 成功后，提取签名信息：

```swift
// 新增代码（在 SecStaticCodeCheckValidity 之前）
var signingInfo: CFDictionary?
SecCodeCopySigningInformation(code, SecCSFlags(rawValue: kSecCSSigningInformation), &signingInfo)
if let info = signingInfo as? [String: Any] {
    let identifier = info[kSecCodeInfoIdentifier as String] as? String ?? "nil"
    let teamId = info[kSecCodeInfoTeamIdentifier as String] as? String ?? "nil"
    let flags = info["flags"] as? UInt32 ?? 0
    GLLog.gkAssess.notice("SecStaticCode signingInfo: \(appName, privacy: .public), id=\(identifier, privacy: .public), team=\(teamId, privacy: .public), flags=\(flags)")
}
```

**预期各 APP 类型的输出**：

| APP 类型 | Code Create | identifier | teamId | flags |
|---------|-------------|-----------|--------|-------|
| A. 未签名 | ❌ 失败 | — | — | — |
| B. Ad-hoc | ✅ 成功 | `com.test.adhoc` 或 `-` | `nil` | adhoc flag |
| C. 篡改签名 | ✅ 成功 | `com.test.damaged` 或 `-` | `nil` | adhoc flag |
| D. 无 quarantine | ✅ 成功 | 同上 | 同上 | — |
| SubLime / CCSwitch | ✅ 成功 | 其 bundle ID | 其 team ID | — |

### 3.3 新增：Quarantine xattr 内容提取

在 `FSEventsWatcher.swift` 的 `handleFSEvent()` 中，在 quarantine 检查通过后读取内容：

```swift
// 新增代码（在 guard Self.hasQuarantine(at: appPath) 之后）
var buffer = [UInt8](repeating: 0, count: 1024)
let size = getxattr(appPath, "com.apple.quarantine", &buffer, 1024, 0, XATTR_NOFOLLOW)
if size > 0 {
    let quarantineValue = String(bytes: buffer[0..<size], encoding: .utf8) ?? "binary"
    GLLog.fsEvents.notice("Quarantine xattr: \(URL(fileURLWithPath: appPath).lastPathComponent, privacy: .public), value=\(quarantineValue, privacy: .public)")
}
```

**Quarantine xattr 格式**：`flag;timestamp;agent_bundle_id;uuid`
- `flag`：`0083` = 用户从网络下载且未通过公证
- `timestamp`：16 进制 Unix 时间戳
- `agent_bundle_id`：下载这个 APP 的浏览器/工具（如 Safari、Chrome）
- `uuid`：唯一标识

### 3.4 USER_MARK 增强：步骤编号化

将 `Cmd+Shift+T` 替换为 7 个独立快捷键：

```swift
// 替换 GreenLightApp.swift 中的 §r04 打点器
CommandGroup(after: .toolbar) {
    ForEach(1...7, id: \.self) { step in
        Button("⏱ Step \(step)") {
            let ts = Int(Date().timeIntervalSince1970 * 1000)
            GLLog.pipeline.notice("⏱ USER_MARK step=\(step) ts=\(ts, privacy: .public) — user action NOW")
        }
        .keyboardShortcut(KeyEquivalent(Character("\(step)")), modifiers: [.command, .shift])
    }
}
```

### 3.5 新增：明文日志备份

新建 `ExperimentLogger.swift`（独立工具类），将关键事件同步写入 `/tmp/greenlight_r07.log`：

```swift
enum ExperimentLogger {
    private static let logFile = "/tmp/greenlight_r07.log"
    
    static func log(_ message: String) {
        let ts = ISO8601DateFormatter().string(from: Date())
        let line = "[\(ts)] \(message)\n"
        if let data = line.data(using: .utf8) {
            if FileManager.default.fileExists(atPath: logFile) {
                if let handle = FileHandle(forWritingAtPath: logFile) {
                    handle.seekToEndOfFile()
                    handle.write(data)
                    handle.closeFile()
                }
            } else {
                FileManager.default.createFile(atPath: logFile, contents: data)
            }
        }
    }
}
```

在各关键节点调用 `ExperimentLogger.log()`，确保即使 `log stream` 断开，数据也不丢失。

### 4.3 「能否确认是谁」的判定逻辑

| 确认维度 | 信息来源 | 可用时机 | 可靠性 |
|---------|---------|---------|--------|
| APP 文件名 | FSEvents 路径 | GK 处理完后（~4-7s） | ⭐⭐⭐⭐⭐ 100% 可靠 |
| Bundle ID | Info.plist 读取 | 同上 | ⭐⭐⭐⭐ 大多数 APP 有 |
| Code Identifier | SecStaticCode 签名 | 同上 | ⭐⭐⭐ 仅签名有效时 |
| Team ID | SecStaticCode 签名 | 同上 | ⭐⭐ 仅正式签名有 |
| syspolicyd 日志中的路径 | LogStream Channel A | GK 处理中（~2s）| ❓ **本次实验验证** |
| Quarantine 来源 | xattr 解析 | 同上 | ⭐⭐⭐ 辅助信息 |

**关键问题**：第 5 行「syspolicyd 日志中的路径」是**唯一可能在 FSEvents 之前到达的身份信息**。r04 因 `<private>` 无法验证。r07 修复后将首次看到 syspolicyd 的原始日志内容。
