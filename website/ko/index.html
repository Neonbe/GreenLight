수정하는 오류 메시지

이런 오류가 보이나요? GreenLight가 자동으로 수정합니다.

이러한 macOS 경고는 앱이 고장났다는 의미가 아닙니다. GreenLight가 실시간으로 감지하고 원클릭으로 수정합니다.

"앱"이(가) 손상되어 열 수 없습니다. 휴지통으로 이동해야 합니다.

macOS Gatekeeper의 격리 플래그(com.apple.quarantine)로 인한 "오탐지"입니다. 앱은 실제로 손상되지 않았습니다. GreenLight가 자동 감지하고 원클릭으로 수정합니다.

"앱"은(는) 개발자를 확인할 수 없기 때문에 열 수 없습니다.

앱이 Apple 공증을 받지 못해 macOS가 개발자 신원을 확인할 수 없습니다. GreenLight로 완전 자동 처리. 설치 후 아무것도 할 필요 없습니다.

Mac을 보호하기 위해 "앱"이(가) 차단되었습니다.

앱에 맬웨어가 포함된 것이 아닙니다. Apple이 검증할 수 없을 뿐입니다. GreenLight가 안전하게 격리 플래그를 제거. 한 번 설치하면 영구 자동화.

❌ 기존 방법 (매번 반복)

  1. 오류 표시 → 당황
  2. 해결 방법 검색
  3. 터미널 열기
  4. xattr -rd com.apple.quarantine /경로/앱.app 입력
  5. 앱 업데이트 → 처음부터 반복

✅ GreenLight 사용 (한 번만)

  1. GreenLight 다운로드 (한 번만)
  2. 앱 차단됨 → 알림 자동 표시
  3. "수정 후 열기" 클릭
  4. 끝. 이후 모두 자동.

GreenLight의 특징

  • ✓ 완전 자동 감지 — 수동 확인 불필요, 시스템 이벤트 실시간 모니터링
  • ✓ 원클릭 수정 — 시스템 알림에서 바로 수정
  • ✓ 영구 무료 — 오픈소스, MIT 라이선스
  • ✓ 프라이버시 우선 — 네트워크 통신 제로, 데이터 수집 제로
  • ✓ 초경량 — 메모리 30MB 미만, CPU 1% 미만

자주 묻는 질문

"손상되어 열 수 없습니다" 오류가 뜨면?

앱이 실제로 손상된 것이 아닙니다. macOS Gatekeeper가 다운로드 파일에 com.apple.quarantine 속성을 부여하여 Apple 공증이 없는 앱을 차단합니다. GreenLight(무료 오픈소스)를 설치하면 자동 감지 및 원클릭 수정이 가능합니다.

터미널 없이 차단된 앱을 여는 방법은?

GreenLight를 설치하세요. 메뉴바에서 조용히 실행되며 Gatekeeper가 앱을 차단하면 즉시 알림을 보냅니다. "수정 후 열기"를 클릭하면 터미널이나 xattr 명령어 없이 해결됩니다.

GreenLight는 안전한가요?

완전히 안전합니다. MIT 라이선스 오픈소스로 네트워크 통신 제로, 데이터 수집 제로. 완전히 로컬에서 실행됩니다.
